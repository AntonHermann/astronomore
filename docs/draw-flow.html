Wie ein Frame entsteht — von den einmalig angelegten GPU-Datenstrukturen über das Pro-Frame-Update der Uniforms bis zum Zeichnen im Render-Pass. Einstiegspunkt ist State in src/lib.rs.

Matrizen

Welche Matrix wo lebt — und wer sie benutzt

Ein Vertex läuft durch zwei GPU-Matrizen: erst die Model-Matrix (Group 2, pro Körper) nach World Space, dann view_proj (Group 1, einmal) nach Clip Space. Alles andere sind CPU-seitige Zwischenmatrizen, die nie einen Buffer sehen.

Space-Transform im Vertex-Shader (shader.wgsl:67–71)

model space ModelGroup 2 · pro Körper→ world space view_projGroup 1 · pro Pass→ clip space

clip = camera.view_proj · (model_to_world · position) — im Shader sind das die einzigen zwei mat4x4-Uniforms. Die Normale nimmt denselben Model-Weg (mit w=0), gilt aber nur, weil die Skalierung uniform ist.

Kernidee: Genau zwei Matrizen leben in GPU-Buffern — Model (Group 2, pro Körper eine, geschrieben in scene.update) und view_proj (Group 1, eine, geschrieben in camera_rig.update). Die Model-Matrix wird pro Körper zusammengesetzt (orbital→world, spin, scale), die View-Projektion wird aus view · projection kombiniert. Alle Faktoren dazwischen sind kurzlebige CPU-Werte.

Phase 0

Datenstrukturen anlegen

State::new() — src/lib.rs:82 · läuft genau einmal beim Start

Alle GPU-Ressourcen, die über die gesamte Laufzeit bestehen bleiben: Buffer, Texturen, Bind-Group-Layouts und Pipelines. Nichts davon wird pro Frame neu erzeugt.

Aufrufkette

  1. GpuContext::new(window)Surface, Device, Queue, Adapter + Depth-Texture — gpu.rs
  2. Texture::bind_group_layout(device)Layout für Sampler + TextureView (Group 0); danach dbg.png laden
  3. Scene::new(device)leere Körperliste + model_bind_group_layout (Group 2)
  4. for def in planets::BODIESje Körper: Textur laden → CelestialBody::new(...) → scene.add_celestial_body(body, parent)
  5. CameraRig::new(...)Kamera-Uniform-Buffer + Bind-Group (Group 1), Projektion
  6. SceneProperties::new(device)Blinn-Phong-Uniform (Group 3)
  7. Pipelines::new(...)baut fill, wireframe, normals, grid — braucht alle 4 Bind-Group-Layouts
  8. Grid- & Arrow-Meshes, EguiLayerHilfsgeometrie, Line-Brightness-Buffer, Identity-Model, UI-Overlay

Was pro Körper entsteht — CelestialBody::new (celestial_body.rs:67)

Mesh::sphere 128×64 → VertexBuffer + IndexBuffer normals_mesh (GridMesh) ModelUniform (4×4 Matrix) model_buffer (UNIFORM｜COPY_DST) model_bind_group (Group 2) Texture (Group 0)

Merke: Der model_buffer wird hier einmal alloziert und später nur noch per queue.write_buffer überschrieben — keine Neu-Allokation im laufenden Betrieb.

Phase 1

Updaten

State::update() — src/lib.rs:224 · jeden Frame vor dem Zeichnen

Nur Daten ändern, nichts zeichnen. Die einzige Wahrheit ist sim.time; aus ihr folgen alle Positionen. Ergebnis jedes Schritts landet per queue.write_buffer in den bereits existierenden Uniform-Buffern.

Aufrufkette

  1. dt = now − last_updatereale Frame-Zeit messen
  2. sim.advance(dt)sim.time += dt · multiplier (nur wenn nicht pausiert) — sim.rs:37
  3. scene.update(sim.time, queue)Kern — siehe unten (scene.rs:65)
  4. camera_rig.update(dt, scene, queue)View·Projection neu berechnen → Kamera-Uniform schreiben
  5. scene_properties.update(queue)Beleuchtungsparameter (Group 3) schreiben
  6. if show_arrows { arrow_mesh.update(...) }Debug-Pfeile neu aufbauen (optional)

Herzstück — scene.update() pro Körper

  1. cb.update(sim_time)Position aus OrbitalModel (Fixed / Parametric / Vsop87) → orbital_transform + spin_transform
  2. world_transform = parent · relativetopologische Iteration (Eltern stehen immer vor Kindern) akkumuliert die Hierarchie
  3. update_model_uniform(world)ModelUniform = world · spin · scale(radius)
  4. queue.write_buffer(model_buffer)neue Matrix in den GPU-Buffer des Körpers kopieren

Warum getrennt von Draw: Pausieren, Zeitraffer und Datumssprünge funktionieren, weil die Orbitlogik nur an sim.time hängt — nie an Instant::now().

Phase 2

Zeichnen

State::render() — src/lib.rs:295 · Command-Encoder → Render-Pass → submit

Aus dem aktuellen Zustand der Buffer entstehen GPU-Befehle. Nichts wird hier mehr berechnet — es wird nur Pipeline gesetzt, Bind-Groups gebunden und gezeichnet.

Aufrufkette

  1. surface.get_current_texture()Ziel-Frame holen (Suboptimal/Outdated → neu konfigurieren)
  2. device.create_command_encoder()sammelt alle Zeichenbefehle
  3. encoder.begin_render_pass()Color-Attachment Clear (Hintergrund) + Depth Clear(1.0)
  4. set_pipeline(fill｜wireframe)Bind-Groups 0–3 setzen (Textur, Kamera, Model, Scene-Props)
  5. draw_scene(...)je Körper: draw_celestial_body → VBO/IBO + Group 0/1/2 → draw_indexed
  6. optional: normals · arrows · gridseigene Pipelines, gesteuert über ViewOptions
  7. 2. Render-Pass: eguiLoadOp::Load — UI wird über die Szene komponiert
  8. queue.submit(encoder.finish()) · present()Befehle an die GPU, Frame anzeigen
